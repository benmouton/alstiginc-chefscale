PHASE 6: My Prices Screen
Build the My Prices screen (app/(tabs)/prices.tsx) for ChefScale.

This is where users manage ingredient costs that apply across all recipes.

LAYOUT:

HEADER:
- "My Prices" title
- Search bar to filter ingredients
- "Last updated" timestamp

INGREDIENT PRICE LIST:
- Grouped by category (Dairy, Protein, Produce, Dry Goods, Spices, Other)
- Each row shows:
  - Ingredient name
  - Current cost: "$4.99 per lb" (or "No price set" in gray)
  - Tap to edit → inline expansion or modal:
    - Purchase price (what you pay): number input
    - Purchase unit (what you buy it in): dropdown (lb, oz, gallon, each, bag, case, can)
    - App auto-calculates cost per recipe unit
  - Number of recipes using this ingredient: "(used in 5 recipes)"

EMPTY STATE:
- "Prices will appear here as you add ingredients to recipes. Tap any ingredient to set its cost."

AUTO-POPULATION:
- When user adds ingredients to any recipe, they automatically appear in this list
- Ingredient names are normalized (lowercase, trimmed) for matching across recipes
- If user sets a price here, it auto-applies to ALL recipes using that ingredient

BULK ACTIONS:
- "Update All Prices" — prompts through each ingredient one by one (useful for weekly price updates from vendor invoices)

CLOUD SYNC (add to existing SQLite logic):
- Create matching tables in Supabase (recipes, ingredients, instructions, ingredient_prices)
- Add user_id column to all tables
- Enable Row Level Security: users can only read/write their own data
- Sync logic in lib/sync.ts:
  - On save: write to SQLite first (instant), then push to Supabase in background
  - On launch: pull any server updates newer than last sync timestamp
